# 日本株カレンダー・アノマリー検証システムのPoC構築 (Live API E2E)

以下の要件定義に従い、Python（Polars + DuckDB + vectorbt）を用いたアノマリー分析システムのPoCを構築してください。
本プロジェクトでは、実際のJ-Quants API（Freeプラン）へ接続し、E2Eで動作検証を行うことを必須とします。

## 必須要件（環境変数とシークレット管理）
1. **クレデンシャルの分離**: J-Quants APIの認証情報（リフレッシュトークン等）はコード内に絶対にハードコードしないでください。
2. **.envの利用**: `python-dotenv` などを利用し、実行環境のルートにある `.env` ファイルから環境変数を読み込む設計としてください。
3. **.env.exampleの作成**: リポジトリにコミットするための雛形として、キー名のみを記載した（値は空、またはダミーの）`.env.example` を必ず作成してください。
4. **セキュリティ**: デバッグ用のprint文やロガーで、取得したトークンやパスワードを出力しないよう厳密にコーディングしてください。

## 開発サイクル指定

### Cycle 1: データパイプラインの構築 (ETL & Storage)
1. **API Ingestion**: 実際のJ-Quants API (Freeプラン) に接続し、取得可能な最新期間（12週間分）の日足データを取得するモジュール。
   - 認証エラーやレートリミットの例外処理（リトライロジックなど）を実装すること。
   - どの日付で実行してもエラーにならないよう、ハードコードされた日付ではなく相対的な期間指定で取得すること。
2. **Transformation**: Polarsを用いて以下の特徴量を生成するモジュール。
   - 曜日フラグ（1〜5）
   - 月初・月末フラグ
   - 前日比リターン、日中リターン、オーバーナイトリターン
3. **Storage**: 処理済みデータを`.parquet`形式でローカルに保存し、DuckDBからクエリ可能な状態にするモジュール。
*期待する成果物: `.env`から認証情報を読み込み、実APIからデータを取得し、Polarsで加工後、Parquetとして保存・読み込みができる一連のスクリプトと、それをパスするPytest（通信を伴うLive Testを含む）。*

### Cycle 2: バックテストと統計検定 (Backtesting & Stats)
1. **Statistical Testing**: Cycle 1で作成したデータに対し、`statsmodels`等を用いて「特定の曜日」のリターン分布が統計的に有意な差を持つかを検証するモジュール。
2. **Backtesting Engine**: Polarsのデータフレームを`vectorbt`が処理可能な形式に変換し、「指定曜日買い・指定曜日売り」のバックテストを実行するモジュール。手数料率やスリッページを考慮すること。
*期待する成果物: 統計検定の結果出力機能と、vectorbtによるシャープレシオや勝率などのパフォーマンス評価指標を出力する機能、およびシステム全体の統合テスト。*
